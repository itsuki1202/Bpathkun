# 実装計画: 分析ページの比較・シミュレーション機能追加

分析ページに、特定の対象（個人・チーム・店舗）を比較し、さらに「あと何件獲得していたらどうなっていたか」をシミュレートできる機能を追加します。

## ユーザーレビューが必要な項目
- **UIの配置**: 既存の「分析」ページの中にタブを追加するか、新しいメニュー項目として「比較・シミュレーション」を追加するか。
- **シミュレーションの範囲**: 「順位」を再計算する場合、全データの再計算が必要になるため、計算負荷や表示速度への影響。

## 提案する変更内容

### 1. 新しいナビゲーションモードの追加
- サイドバーに「比較・シミュレーション」を追加、または既存の「分析」ページ内に「比較タブ」を新設します。

### 2. 比較・シミュレーション UI (app.py)
- **対象選択**: 
    - 比較単位の選択（個人、チーム、店舗）。
    - 比較対象Aと比較対象Bの選択。
- **比較表示**:
    - AさんとBさんの各項目実績・スコアを並列で表示。
    - レーダーチャート等による視覚的な差異の表示。
- **シミュレーションパネル**:
    - 対象Aに対して、「項目Xをあと●件獲得していたら」という入力フォーム（スライダーや数値入力）を提供。
- **シミュレーション結果表示**:
    - 「あと●件獲得していたら、合計得点は●点（現在より+●点）になり、順位は●位（現在より●位アップ）になっていた」というテキストおよび数値を表示。

### 3. 計算ロジック (utils/calculation.py または app.py 内)
- **仮想データの作成**: 入力された「追加件数」を現在の実績に加算した、対象Aの仮想レコードを作成。
- **再計算**: 仮想レコードを含めた全体データに対して得点計算とランキング計算を再度実行。
- **差分抽出**: 元の順位・得点と、仮想結果の差分を計算。

## 検証計画

### 自動テスト
- 現在の計算ロジック（`calculate_scores`）に仮想データを渡した際に、正しく得点と順位が計算されるかを `tests/` 内にテストスクリプトを作成して検証（検討中）。

### 手動検証
1. 「比較・シミュレーション」メニューに移動。
2. 「個人」を選択し、スタッフAとスタッフBを選択。
3. スタッフAの特定の項目（例：HS新規）を +5 件にする。
4. 「シミュレーション実行」を押し、表示される順位・得点が期待通りに変動することを確認。
5. 同様の操作を「チーム」「店舗」でも行い、集計値が正しく連動することを確認。
